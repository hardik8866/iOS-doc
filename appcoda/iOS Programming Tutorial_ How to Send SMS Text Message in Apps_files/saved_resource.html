                    Kingsley Sendze Wouter â€¢ a year ago

                    Resolved it. I removed the implementation of the custom Navigation bar from my delegate.m (in didFinishLaunchingWithOptions) such that the custom bar was not across the entire app. An in my viewWillAppear method for each of my tab bar view controllers where I wanted the customs, I added:
                    //custom navBar implementing within each tab bar:
                    if ([[UIDevice currentDevice] userInterfaceIdiom] == UIUserInterfaceIdiomPad) {
                    //custom navigation bar:
                    [[UIApplication sharedApplication] setStatusBarStyle:UIStatusBarStyleLightContent]; //changes bateery bar, time etc to white

                    [self.navigationController.navigationBar setBackgroundImage:[UIImage imageNamed:@"navBar.jpg"] forBarMetrics:UIBarMetricsDefault];
                    } else {

                    //custom navigation bar:
                    [self.navigationController.navigationBar setBackgroundImage:[UIImage imageNamed:@"navBarIPhone.jpg"] forBarMetrics:UIBarMetricsDefault];

                    //iOS 5, 6, 7
                    [[UIApplication sharedApplication] setStatusBarStyle:UIStatusBarStyleLightContent]; //changes bateery bar, time etc to white

                    }

                    This leaves the navbar defaulting to the out of the box and I choose which view controllers I want to apply the custom navbars to.

                                                                                Yasin Abdul mohamed â€¢ a year ago

                                                                                The MFMessageComposeViewController wants the attachment to have the correct extension for the type of image you're uploading. I verified by testing with a PNG file, and the following variations of adding the attachment data:

                                                                                [messageController addAttachmentData:imgData typeIdentifier:@"public.data" filename:@"image"];
                                                                                [messageController addAttachmentData:imgData typeIdentifier:@"public.data" filename:@"image.abc"];
                                                                                [messageController addAttachmentData:imgData typeIdentifier:@"public.data" filename:@"image.png"];

                                                                                Only the last option worked. I didn't need to change the typeIdentifier, although it probably would make sense to choose a UTI that matches the type of data.
